**Pod не создался (status: Pending)**

Описание проблемы: под не может быть назначен на узел из-за нехватки ресурсов (CPU, память) или других ограничений.

Диагностика:

```
# Проверка статуса пода
kubectl get pod <pod-name> -o wide

# Детальная информация и события
kubectl describe pod <pod-name> 
```

Пример вывода в Events:

```
Events:
  Type     Reason            Age   From               Message
  ----     ------            ----  ----               -------
  Warning  FailedScheduling  2m    default-scheduler  0/3 nodes are available: 3 Insufficient cpu.
  Warning  FailedScheduling  1m    default-scheduler  0/3 nodes are available: 2 Insufficient memory, 1 node(s) had taint {node-role.kubernetes.io/master: }, that the pod didn't tolerate. 
```

Проверка ресурсов на узлах:

```
# Просмотр доступных ресурсов на узлах
kubectl describe nodes | grep -A 5 "Allocated resources" 
```

Как чинить:

```
# Вариант 1: Уменьшить запросы ресурсов в поде
kubectl edit pod <pod-name>
# Изменить resources.requests.cpu и resources.requests.memory

# Вариант 2: Добавить новые узлы в кластер (для облачных провайдеров)
kubectl scale nodes --replicas=4  # Для managed clusters

# Вариант 3: Освободить ресурсы, удалив ненужные поды
kubectl delete pod <unnecessary-pod>

# Вариант 4: Добавить tolerations для master-узлов (не рекомендуется для production)
kubectl edit pod <pod-name>
# Добавить:
# tolerations:
# - key: node-role.kubernetes.io/master
#   operator: Exists
#   effect: NoSchedule 
```

**Cтатус пода `CrashLoopBackOff`**

Описание проблемы: приложение внутри контейнера падает сразу после старта.

Диагностика:

```
# Просмотр текущих логов
kubectl logs <pod-name>

# Просмотр логов предыдущего запуска (после рестарта)
kubectl logs <pod-name> --previous

# Для многоконтейнерных подов
kubectl logs <pod-name> -c <container-name>

# Проверка статуса и причины рестартов
kubectl describe pod <pod-name> | grep -A 10 "Containers:" 
```

Пример вывода describe:

```
Containers:
  app:
    State:          CrashLoopBackOff
    Last State:     Terminated
      Reason:       Error
      Exit Code:    1
      Started:      Mon, 01 Jan 2024 10:00:00 +0000
      Finished:     Mon, 01 Jan 2024 10:00:05 +0000
    Restart Count:  5 
```

Как чинить:

```
# Вариант 1: Исправить команду запуска
kubectl edit pod <pod-name>
# Проверить и исправить spec.containers[].command и spec.containers[].args

# Вариант 2: Проверить переменные окружения
kubectl set env pod/<pod-name> --list
kubectl set env pod/<pod-name> DATABASE_URL=postgres://correct-url

# Вариант 3: Запустить контейнер в режиме отладки
kubectl run debug-pod --image=<your-image> --command -- sleep 3600
kubectl exec -it debug-pod -- /bin/bash
# Вручную запустить приложение и посмотреть ошибки

# Вариант 4: Изменить entrypoint для отладки
kubectl patch pod <pod-name> --type='json' -p='[{"op": "replace", "path": "/spec/containers/0/command", "value": ["sleep", "3600"]}]'
# После этого можно зайти в контейнер и отладить

# Вариант 5: Проверить права доступа и файловую систему
kubectl exec <pod-name> -- ls -la /app
kubectl exec <pod-name> -- whoami 
```
